## 1. Overview

The backend is a **RESTful API** built with **FastAPI** (Python 3.14). It serves as the central data layer for the TTC-IPMS system, providing:

- JWT-based authentication and role-based authorization
- Full CRUD operations for all IP management entities
- Automatic database schema creation via SQLModel on startup
- Seeded demo data for development

The API is consumed exclusively by the React frontend (SPA). Documentation is auto-generated by FastAPI at `/docs` (Swagger UI) and `/redoc`.

## 6. Security & Authentication

### Password Hashing (`app/core/security.py`)

Uses **bcrypt** via `passlib`:

```python
pwd_context = CryptContext(schemes=["bcrypt"], deprecated="auto")

def hash_password(password: str) -> str:
    return pwd_context.hash(password)

def verify_password(plain: str, hashed: str) -> bool:
    return pwd_context.verify(plain, hashed)
```

### JWT Token Generation

```python
def create_access_token(subject: str, extra: Optional[dict] = None) -> str:
    to_encode = {"sub": subject}  # subject = user email
    if extra:
        to_encode.update(extra)   # adds role to token payload
    expire = datetime.utcnow() + timedelta(minutes=settings.access_token_expire_minutes)
    to_encode["exp"] = expire
    return jwt.encode(to_encode, settings.secret_key, algorithm=settings.algorithm)
```

### Current User Dependency (`app/core/deps.py`)

A FastAPI dependency injected into protected endpoints:

```python
# Usage in any router:
current_user: User = Depends(get_current_user)
```

Decodes the Bearer token from the `Authorization` header, verifies the signature, and returns the `User` from the database.

### Auth Endpoints (`app/routers/auth.py`)

| Method | Path | Body | Response | Description |
|---|---|---|---|---|
| `POST` | `/auth/register` | `UserCreate` JSON | `UserRead` | Register new user |
| `POST` | `/auth/login` | `OAuth2PasswordRequestForm` (form data) | `{ access_token, token_type }` | Login and get JWT |

> **Note:** Login uses `OAuth2PasswordRequestForm` — the frontend must send `Content-Type: application/x-www-form-urlencoded` with `username` (email) and `password` fields.

## 7. Database Layer

**[`app/database.py`](file:///c:/Users/ASUS/Documents/GitHub/technology-transfer-cell/backend/app/database.py)**

### Engine

```python
engine = create_engine(settings.database_url, echo=False)
```

Connects to the PostgreSQL database defined in config.

### Session Dependency

```python
def get_session():
    """Dependency generator to provide a database session for requests."""
    with Session(engine) as session:
        yield session
```

Used in routers as:
```python
session: Session = Depends(get_session)
```

Each request gets its own isolated database session that is automatically closed after the request.

### Database Initialization (`init_db`)

Called automatically at application startup via the `lifespan` context manager:

1. Imports all model classes (to register them with SQLModel metadata)
2. Calls `SQLModel.metadata.create_all(engine)` — creates all tables if they don't exist
3. Calls `seed_data(session)` — inserts default data if tables are empty

## 13. Running the Backend

### Prerequisites

- Python ≥ 3.14
- PostgreSQL running (via Docker or locally)
- `uv` package manager

### Using Docker for PostgreSQL

```bash
# Start the PostgreSQL container
docker-compose up -d

# This starts PostgreSQL on port 5432
# Database: ttc_db | User: postgres | Password: postgrespassword
```

### Install Dependencies

```bash
cd backend

# Using uv (recommended)
uv sync

# Or using pip
pip install -e .
```

### Start the Server

```bash
# Development mode with auto-reload
uv run uvicorn app.main:app --reload --port 8000

# Or with standard uvicorn
uvicorn app.main:app --reload --port 8000

# API available at:
# http://localhost:8000
# http://localhost:8000/docs  ← Swagger UI
# http://localhost:8000/redoc ← ReDoc
# http://localhost:8000/health ← Health check
```

> On first startup, `init_db()` automatically creates all database tables and seeds demo data.
